Critical Rendering Path

The browser wants to show you the page as fast as possible.

  1. Loads the critical resources (HTML, CSS, JS needed for first screen).
  2. Builds DOM and CSSOM.
  3. Creates Render Tree.
  4. Paints the page quickly so you see something (First Contentful Paint).
  5. Loads other stuff in the background.

Performance Metrics

  • TTFB: Time to First Byte – how fast the server responds.
  • FCP: First Contentful Paint – when first content appears.
  • LCP: Largest Contentful Paint – when biggest part is loaded.
  • TTI: Time to Interactive – when page is usable.
  • CLS: Cumulative Layout Shift – how much things jump around.
