Introduction to CSS and Rendering Pipeline

Browsers parse HTML and CSS to build internal structures before painting pixels on the screen.

This process involves three key trees:

DOM Tree (Document Object Model)

The DOM tree represents the HTML document structure as nodes.

CSSOM Tree (CSS Object Model)

CSSOM represents the parsed CSS rules and declarations as a tree.

Render Tree

Render Tree combines DOM and CSSOM to represent styled visible elements for painting.

The CSS Cascade and Priority

The cascade resolves style conflicts using priority rules and specificity.

Priority Order Visualization

  • Inline styles (highest priority)
  • #id selectors
  • .class, attribute, pseudo-class selectors
  • element and pseudo-element selectors

Example: Conflict Resolution

HTML:

<h1 id="title" class="highlight" style="color: blue;">Hello</h1>

CSS:

#title { color: red; }
.highlight { color: green; }
h1 { color: orange; }

Which color applies? style="color: blue;" wins because inline styles have highest priority.

CSS Selectors and Specificity

Selectors target elements in HTML. Specificity determines which rule wins if multiple apply.

Specificity values:

  • Inline styles: 1000 points
  • ID selectors: 100 points
  • Classes, attributes, pseudo-classes: 10 points
  • Elements, pseudo-elements: 1 point

The rule with higher specificity wins; if equal, the later declared rule wins.
