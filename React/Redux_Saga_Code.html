Redux Saga Flow: Step-by-Step Interactive Visualization

1. Watcher Saga ?

The watcher saga listens for specific Redux actions using effects like takeEvery or takeLatest and calls the worker saga in response. 

function* watchFetchData() {
  yield takeEvery('FETCH_REQUESTED', fetchData);
}
2. Worker Saga ?

The worker saga performs the actual side effect, such as making an API call, and then dispatches success or failure actions. 

function* fetchData(action) {
  try {
    const data = yield call(Api.fetchUser, action.payload.userId);
    yield put({ type: 'FETCH_SUCCEEDED', data });
  } catch (e) {
    yield put({ type: 'FETCH_FAILED', message: e.message });
  }
}
3. Redux Saga Effects ?

Redux Saga provides effects to manage side effects declaratively:

  • call(fn, ...args): calls a function and waits for the result.
  • put(action): dispatches an action.
  • take(actionType): pauses saga until action is dispatched.
  • fork(fn, ...args): non-blocking call to a saga. 
yield call(api.fetchData);
yield put({ type: 'ACTION_TYPE' });
yield take('ACTION_TYPE');
yield fork(workerSaga);
4. Saga Middleware Integration ?

The saga middleware is created and connected to the Redux store, then the root saga is run. 

import createSagaMiddleware from 'redux-saga';
const sagaMiddleware = createSagaMiddleware();
const store = createStore(
  reducer,
  applyMiddleware(sagaMiddleware)
);
sagaMiddleware.run(rootSaga);
5. Root Saga ?

The root saga aggregates multiple watcher sagas using all effect. 

function* rootSaga() {
  yield all([
    watchFetchData(),
    watchAnotherAction(),
  ]);
}
