üìö What is Sliding Window?

Core Concept:

A Sliding Window is a technique where you maintain a contiguous subarray/substring that "slides" through the input. Instead of recalculating values for every possible window from scratch, you cleverly add elements from the right and remove from the left.

üé¨ Visual Example: Moving Through an Array

Array: [1, 3, 2, 6, 4, 1, 2, 5, 3]
Window Size: 3

Step 1: Initial window [1, 3, 2] ‚Üí Sum = 6
Step 2: Slide right (remove 1, add 6) ‚Üí [3, 2, 6] ‚Üí Sum = 11
Step 3: Slide right (remove 3, add 4) ‚Üí [2, 6, 4] ‚Üí Sum = 12
And so on...

üîë Key Insight: Why It Works

‚ùå Brute Force

for i in range(n): for j in range(i, i+k): calculate_sum()
Time: O(n*k)

‚úÖ Sliding Window

sum = calculate_first_k() for i in range(k, n): sum = sum - arr[i-k] + arr[i]
Time: O(n)

üìä Types of Sliding Window

There Are 2 Main Types

üì¶ Fixed Window Size

Window size = K (constant)

Size NEVER changes. You just slide K elements at a time across the array.

Example: Find max sum of exactly 3 consecutive numbers

üìà Dynamic Window Size

Window size varies based on condition

Size CHANGES. Expand when condition satisfied, shrink when violated.

Example: Find longest substring with no repeating chars

Detailed Comparison

Aspect Fixed Window Dynamic Window
Size Constant (K) Varies based on condition
Setup Pre-calculate first K elements Initialize left and right pointers
Sliding Logic Simple: remove left, add right Complex: expand right, shrink left via while
Difficulty ‚≠ê‚≠ê Easy ‚≠ê‚≠ê‚≠ê Medium-Hard

‚è∞ When to Use Sliding Window

‚úÖ Use Sliding Window When:

  • Problem asks for longest/shortest substring/subarray with a condition
  • Problem asks for maximum/minimum sum of consecutive elements
  • Problem involves K consecutive elements
  • Problem asks to count subarrays with specific property
  • You need to find contiguous subsequences (not scattered)
  • You can efficiently update window state by adding/removing one element

‚ùå Don't Use Sliding Window When:

Looking for non-contiguous elements
Example: "Find any two elements that sum to 10"
Problem requires sorting or rearranging
Example: "Merge K sorted lists"
Window state is complex and can't be updated efficiently
Example: "Find K-smallest elements"

üí° Sample Problems: Brute ‚Üí Better ‚Üí Optimal

Problem 1: Maximum Sum of K Consecutive Elements

Find the maximum sum of any K consecutive elements in an array.

‚ùå Approach 1: Brute Force

function maxSumBrute(arr, k) { let maxSum = 0; // Try every possible window of size k for (let i = 0; i <= arr.length - k; i++) { let sum = 0; // Calculate sum for this window for (let j = i; j < i + k; j++) { sum += arr[j]; } maxSum = Math.max(maxSum, sum); } return maxSum; } // Time: O(n*k) - SLOW! ‚ùå // Space: O(1)

‚ö° Approach 2: Better (Sliding Window)

function maxSumBetter(arr, k) { // Step 1: Calculate sum of first window let currentSum = 0; for (let i = 0; i < k; i++) { currentSum += arr[i]; } let maxSum = currentSum; // Step 2: Slide the window - remove left, add right for (let i = k; i < arr.length; i++) { // Remove leftmost element of previous window currentSum -= arr[i - k]; // Add rightmost element of new window currentSum += arr[i]; maxSum = Math.max(maxSum, currentSum); } return maxSum; } // Time: O(n) - FAST! ‚úÖ // Space: O(1)

‚ú® Approach 3: Optimal (Same as Better)

For this problem, the "better" approach IS the optimal approach! The sliding window technique is already O(n) which is the best we can do (we must visit each element at least once).

Problem 2: Longest Substring Without Repeating Characters

Find the length of the longest substring without repeating characters.

‚ùå Approach 1: Brute Force

function lengthOfLongestSubstringBrute(s) { let maxLen = 0; // Check every possible substring for (let i = 0; i < s.length; i++) { for (let j = i + 1; j <= s.length; j++) { const substring = s.substring(i, j); // Check if all chars are unique if (new Set(substring).size === substring.length) { maxLen = Math.max(maxLen, substring.length); } } } return maxLen; } // Time: O(n¬≥) - VERY SLOW! ‚ùå // Space: O(min(m,n))

‚ö° Approach 2: Better (Optimized Brute)

function lengthOfLongestSubstringBetter(s) { let maxLen = 0; const charSet = new Set(); let left = 0; for (let right = 0; right < s.length; right++) { // If duplicate found, remove from left while (charSet.has(s[right])) { charSet.delete(s[left]); left++; } charSet.add(s[right]); maxLen = Math.max(maxLen, right - left + 1); } return maxLen; } // Time: O(n) - FAST! ‚úÖ // Space: O(min(m,n))

‚ú® Approach 3: Optimal (Even Better)

function lengthOfLongestSubstringOptimal(s) { let maxLen = 0; const charIndex = {}; // Store last seen index let left = 0; for (let right = 0; right < s.length; right++) { // If char seen before and in current window, // move left pointer directly to after the duplicate if (s[right] in charIndex && charIndex[s[right]] >= left) { left = charIndex[s[right]] + 1; } charIndex[s[right]] = right; maxLen = Math.max(maxLen, right - left + 1); } return maxLen; } // Time: O(n) - FASTEST! ‚úÖ‚úÖ // Space: O(min(m,n)) - efficient
